# ERD Diagram Update: Schema 7.22 → 7.24

**Date:** 2026-07-09  
**Status:** Approved (revised 2026-07-11 — regeneration approach)

## Goal

Produce five new 7.24 ER diagram SVGs, grouped the same way as the existing 7.22 diagrams (BPMN engine, DMN engine, CMMN engine, History, Identity), generated from `docs/documentation/reference/db-schema/schema.mermaid` (the current 7.24 source of truth). Update `index.md` to reference the new files.

## Revision Note

The original design called for byte-level SVG surgery on the existing 7.22 SVGs to preserve their exact Inkscape-generated layout. Investigation showed those SVGs encode all text as vector glyph paths (`<use xlink:href="#glyphF-N">` with no font/space metadata, and non-shared glyph IDs across files), making reliable decoding impractical within reasonable effort. Decided instead to regenerate the diagrams from the mermaid source using `@mermaid-js/mermaid-cli` (`mmdc`, available via `npx`, already a project dependency — see `package.json`). This produces accurate, real-`<text>`-based SVGs. Visual styling will differ from the 7.22 originals (mermaid's built-in ER diagram theme, using the `classDef` colors already present in `schema.mermaid`), but content is guaranteed correct and the generation is scriptable for future schema versions.

## SVG Files

Target: `static/img/documentation/user-guide/process-engine/database/erd_724_{bpmn,dmn,cmmn,history,identity}.svg`  
(7.22 originals are left untouched.)

## Table Groupings

Derived from table prefix/domain semantics (verified against `schema.mermaid`):

- **BPMN**: `ACT_GE_PROPERTY`, `ACT_GE_BYTEARRAY`, `ACT_GE_SCHEMA_LOG`, `ACT_RE_DEPLOYMENT`, `ACT_RE_PROCDEF`, `ACT_RE_CAMFORMDEF`, `ACT_RU_EXECUTION`, `ACT_RU_JOB`, `ACT_RU_JOBDEF`, `ACT_RU_TASK`, `ACT_RU_IDENTITYLINK`, `ACT_RU_VARIABLE`, `ACT_RU_EVENT_SUBSCR`, `ACT_RU_INCIDENT`, `ACT_RU_AUTHORIZATION`, `ACT_RU_FILTER`, `ACT_RU_METER_LOG`, `ACT_RU_TASK_METER_LOG`, `ACT_RU_EXT_TASK`, `ACT_RU_BATCH`
- **DMN**: `ACT_RE_DECISION_DEF`, `ACT_RE_DECISION_REQ_DEF`
- **CMMN**: `ACT_RE_CASE_DEF`, `ACT_RU_CASE_EXECUTION`, `ACT_RU_CASE_SENTRY_PART`
- **History**: all `ACT_HI_*` tables (18 tables)
- **Identity**: all `ACT_ID_*` tables (6 tables)

Relationships are included in a sub-diagram only when both endpoints are in that group (matches the original diagrams' scope note: "diagrams do not visualize implicit connections between tables").

## Implementation Plan

### Step 1 — Mermaid parser + sub-diagram generator

Script `scripts/generate_erd_diagrams.py`:
1. Parse `schema.mermaid`: entity blocks (`ACT_\w+ { ... }`), relationship lines (`TABLE }o--|| TABLE : "COL_"`), and `class TABLE domaincode` assignments.
2. For each of the 5 named groups, build a new `.mmd` string containing: the same `%%init%%` theme header and `classDef` lines, only the entity blocks in that group, only relationships where both endpoints are in-group, and `class` assignments for in-group tables.
3. Write each `.mmd` to a temp/output location, then invoke `npx @mermaid-js/mermaid-cli` (`mmdc -i <file>.mmd -o <file>.svg -b transparent`) to render.
4. Move/rename rendered output to `static/img/documentation/user-guide/process-engine/database/erd_724_<group>.svg`.

### Step 2 — Wiring

In `docs/documentation/user-guide/process-engine/database/database-schema/index.md`, replace the five `erd_722_*.svg` image references with `erd_724_*.svg`.

### Step 3 — Visual check

Open each generated SVG (or the built docs page) in a browser and confirm all expected tables/columns/relationships are present and legible.

## Scope Boundaries

- 7.22 SVGs and their references elsewhere (e.g. `docs/documentation/installation/database-schema.md` if it also references them) are untouched unless explicitly part of this task.
- No attempt to visually match the 7.22 Inkscape style; mermaid's default ER theme (colored via the existing `classDef`s) is acceptable.
- The script is a one-shot generation tool; no need to wire it into the build pipeline unless requested later.

## Success Criteria

1. `erd_724_*.svg` files render correctly in browser and contain real, selectable/searchable text.
2. Every table in each group's SVG matches its `schema.mermaid` column list exactly (name, type, PK/FK marker).
3. `index.md` references point to the new files.
4. No table is missing from all 5 diagrams combined, and no table appears in the wrong group.
